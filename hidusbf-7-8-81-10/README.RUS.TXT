1. Краткое описание.
  Этот драйвер/программа предназначены для "разгона" USB мышей
  под версиями Windows от 98 до XP x64, Windows Vista 32 bit,
  Windows Vista 64 bit (с отключенной проверкой подписи драйверов),
  Windows 7, 8, 8.1, 10 (в x64 - с применением подписанного для Test 
  Mode драйвера с предварительным включением этого самого Test Mode).
  Так же может использоваться для уменьшения частоты опроса
  (и, в отличие от разгона, уменьшение должно работать всегда).

  Примечание - разгон Low speed USB устройств может не работать 
  на устройствах контролируемых немискрософтовским USB стеком драйверов,
  к примеру при использовании USB 3.x контроллера и стороннего драйвера
  для него под Windows 7. Или на очень новых микрософтовских драйверах.

  Программа изменяет частоту опроса исключительно мыши, не затрагивая
  прочие устройства на USB.

  Более подробно с теорией и описанием программы можно ознакомиться по
  адресу: 
  http://www.overclockers.ru/lab/15925/Razgon_USB_myshi_ili_Software_nanosit_otvetnyj_udar.html

  Лично мной проверены Windows 98SE, Windows 2000 SP2/SP4, 
  Windows XP RTM/SP1/SP2/SP3, Windows Vista 32 bit RTM, 
  Windows Vista 64 bit RTM, Windows 7 SP1 x86, Windows 7 SP1 x64,
  Windows 8 x64.
  Тестирование под Windows XP x64 SP1 выполнил Dark_Cloud.
 
2. Установка.
  1. Уменьшение частоты (любых устройств на любях драйверах) или разгон
  Full Speed USB устройств возможен всегда, при этом патчить системный 
  код нет необходимости.

  Если вы испозьзуете программу именно для этих целей - используйте
  драйверы в каталогах DRIVER\NTX86\NOPATCH\ и DRIVER\AMD64\NOPATCH\ 
  заменив ими драйверы в каталогах DRIVER\NTX86\ и DRIVER\AMD64\ соответственно.

  Поскольку под Windows 2000 ограничений в системном коде нет вообще (в т.ч.
  и для разгона Low Speed USB устройств) - использовать драйвер мз каталога
  DRIVER\NTX86\NOPATCH\ рекомендуется всегда. Для Windows 98 и ME рагон
  так же возможен всегда, так что там вообще нет версии с патчем.

  2. Для версий операционных систем начиная с Windows XP для успешного
  разгона Low Speed USB устройств ТРЕБУЕТСЯ наличие оригинальной (неизмененной)
  версии системного драйвера USBPORT.SYS (%sytemroot%\system32\drivers\usbport.sys)
  Начиная с Windows 8 так же работает и на USBXHCI.SYS
  (%sytemroot%\system32\drivers\usbxhci.sys)

  3. Установка моего драйверa HIDUSBF.SYS. Для для этого зайти в SETUP.EXE 
  (в каталоге с ним должны быть .INF и .SYS). Нажать Install Service.

  В случае неустановки сервиса драйвера через SETUP.EXE
  (кнопка Install Service) необходимо установить сервис драйвера 
  средствами ОС: правая кнопка мыши на файле HIDUSBF.INF -> Установить
 
  Если вы хотите разогнать мышь в составе композитного (составного)
  USB устройства - используйте чекбокс "Process Parent". Учтите, что при этом
  разгоняются все устройства в составе композитного, не только мышь.

  Затем выбрать строчку с мышью и поставить птичку Filter On Device. 
  Убедиться что в колонке Filter для мыши стоит Yes. 
  Далее выбрать желаемую частоту. После этого нажать Restart.
  Проверить, что частота изменилась (например программой Mouse Rate Checker).

  Примечание - под x64 версиями Windows Vista и выше вы должны сделать
  HIDUSBF.SYS загружаемым перед рестартом устройства или оно
  прекратит работать! В этом случае внимательно прочитате раздел 3.3 или
  прекратите использования драйвера путём выключения чекбокса Mice Only 
  (для отображения всех устройств, в том числе и нестартующих) 
  + затем кнопка Restart в SETUP.EXE .

  Если частота все же не изменилась, то либо перегрузиться, либо выдернуть 
  и вставить мышь, либо остановить и запустить ее в Device Manager.

  Если частота не увеличивается выше 125 герц, то попробовать УМЕНЬШИТЬ ее
  до 31 или 62 герц. Если частота уменьшается - то следовательно
  драйвер функционирует правильно, однако либо мышь не разгонябельна
  либо USBPORT.SYS/USBXHCI.SYS неоригинальный или слишком новый и поэтому 
  неизвестен программе.

3. Удаление.
  1. Уберите привязку драйвера к мыши через SETUP.EXE (Filter = No)
  2. Для удаления сервиса и файла фильтра:
  правая кнопка мыши на файле HIDUSBFU.INF -> Установить
  3. Если вы удалили драйвер не отвязав его от мыши, то
  мышь перестанет работать. Для возвращения ее в рабочее
  состояние достаточно удалить неработающее устройство,
  соответствующее мыши в Диспетчере устройств (Device Manager),
  а затем переопределить оборудование.
